Classroom Simulator

This was my final project for the Software Design course; one of the options was to create a program which implemented 5 features from a list.
The features I chose were:

  • Maintain the single responsibility principle by having multiple classes that each do only one thing
  • Use at least one interface in your project design
  • Use the decorator pattern to extend a class
  • Implement multiple web pages or Windows forms
  • Present your work-in-progress at the CIS Showcase
I decided to create a simulator of walking through Tower Hall (where I have had all but one of my in-person classes) as classes are in session.
Each page represents a floor of the building; when you click on the tab for a floor in the navigation bar,
the page calls a method to generate random text from an array.
The unique rooms on each floor have their own classes, while the classrooms share four classes,
since all of them do the same thing for the sake of simplicity.
Each classroom generates an activity, the student attendance expressed as a percentage,
an analogy the professor uses during the lecture, and the students' general reaction to that analogy.
